Mecanismo de envío

Para garantizar la entrega íntegra del archivo, TWS combina dos llamadas de sistema clásicas: stat() para conocer el tamaño antes de generar las cabeceras, y un bucle de read()/send() en buffers (típicamente 4096 u 8192 bytes) hasta vaciar el descriptor.

  • stat() publica el tamaño exacto del recurso.
  • La cabecera Content-Length impide cierres prematuros del cliente.
  • El bucle continúa mientras read() devuelva > 0.
  • El uso de buffers fijos limita la memoria por hilo.
  • Si send() retorna menos bytes de los pedidos, se reintenta el remanente.

Esquema del flujo

Fragmento conceptual del envío del archivo, ejecutado en el hilo de la conexión:

struct stat st; stat(path, &st); snprintf(hdr, 256, "HTTP/1.1 200 OK\r\n" "Content-Type: text/plain\r\n" "Content-Length: %ld\r\n\r\n", st.st_size); send(sock, hdr, strlen(hdr), 0); int fd = open(path, O_RDONLY); char buf[4096]; ssize_t n; while ((n = read(fd, buf, sizeof(buf))) > 0) send(sock, buf, n, 0); close(fd); // EOF · 1MB entregado
